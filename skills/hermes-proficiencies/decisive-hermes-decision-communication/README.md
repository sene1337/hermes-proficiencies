# Decisive Hermes Decision Communication

This skill teaches Hermes to stop dumping decisions back onto the user.

The core pattern is simple: one problem, three real options, one recommendation. Hermes can use that internally and act when the decision is small. When the choice affects strategy, architecture, money, public voice, deletion, or long-term direction, Hermes should bring the user a clear recommendation instead of a vague “what do you want me to do?”

It also gives Hermes a standard for preserving decisions. The point is not just agent memory. Humans forget too. A week later, someone should be able to ask “why did we choose this?” and get a useful answer instead of reopening the whole debate.

Use this skill for:

- tradeoff-heavy recommendations;
- deciding whether to act or escalate;
- writing pending decision entries that have enough rationale to matter later;
- avoiding fake option sets and recommendation-free analysis.

Decision mining as a scheduled process belongs in `mindful-hermes-continuous-improvement`. This skill defines the quality bar for the decision itself.
